SELECT

Description

Denotes which fields the SQL query will return.

Use Case

SELECT state_name;
=> returns the state_name field for all records.

FROM

Description

Determines the table from which your are extracting data

Use Case

SELECT state_name
FROM states;
=> extracts state_names from the states table

WHERE

Description

Sets conditions for values in a field whereby if it passes the field for that record will be displayed

Use Case

SELECT state_name
FROM states
WHERE state_name is not null;
=> extracs state_names form the states table for all states whose name aren't null.

ORDER BY

Description

Order your results in ascending order based on the contents of a certain field. Use "ORDER BY field DESC" to order your results in descnding order.

Use Case

SELECT state_name
FROM states
ORDER BY region_id;
=> Orders the records extracted by their region_id.

LIKE

Description

Allows you to extract records that match certain criteria. Often used with the WHERE clause. If I want to find all titles of an auction that contain shirt i.e. WHERE auction.title like ('%Shirt%').

Use Case

SELECT state_name
FROM states
WHERE state_name LIKE ('w%');
=> returns state_names that start with "w"

DISTINCT

Description

Allows you to select the unique values from a sample. This can come in handy when paired with count, if you wanted to count the number of unique users in a certain query you can write count(distinct(user.id))

Use Case

SELECT DISTINCT city_name
FROM customers;
=> Returns a list of the cities that customers are from without repeats.

JOIN

Description

INNER JOIN: Returns all rows when there is at least one match in BOTH tables
LEFT JOIN: Return all rows from the left table, and the matched rows from the right table
RIGHT JOIN: Return all rows from the right table, and the matched rows from the left table
FULL JOIN: Return all rows when there is a match in ONE of the tables

Use Case

SELECT regions.region_name, states.state_name
FROM states
INNER JOIN regions
ON states.region_id=regions.id
=> Returns a table with with region_name and
based on a combined table made up of the states
table and the regions table.

GROUP BY

Description

Aggregates results based on field. Meaning for every instance of a value in a field, the resulting table has another tally

Use Case

TABLE:
ID SHIPPING_COMPANY
1 FedEx
2 UPS
3 UPS
4 USPS
5 FedEx
6 FedEx

SELECT Count(ID), SHIPPING_COMPANY
FROM TABLE
GROUP BY SHIPPING_COMPANY;

=>
1 USPS
2 UPS
3 FedEx

WITH

Description

WITH: Sometimes you need to create a query and have another query pull from those results.

Use Case

WITH;

HAVING

Description

The same as WHERE for aggregate functions.

Use Case

HAVING;

COUNT

Description

count: used to count how many times a result pops up i.e.: SELECT count(user.id) from users - gives you a count of all users.

Use Case

COUNT;

COALESCE

Description

COALESCE: Allows you to return the the value of something, but if there's no value return something else, like 0.

Use Case

COALESCE;

LIMIT

Description

LIMIT: limit the number of results, can often be used to get a small sample of large data.

Use Case

LIMIT;
